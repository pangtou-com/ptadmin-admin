#### 功能说明
> - 从数据库模型中读取数据，如图文，资源下载，产品
> - 注意事项
    >   - 1、指令默认获取归档数据（统一内容管理的数据）
#### 指令参数
- cid
    - 说明：
        - 读取指定栏目信息
        - 可设置读取多个栏目信息，使用【|】分割
        - 默认情况下只获取指定栏目ID数据（通过设置参数：【】开启获取下级数据）
    - 是否必须：否
    - 默认值：空 ｜ 列表页，详情页面时为当前内容的栏目ID
    - 示例： 
  ```
  # 单个支持
  cid = "1"
  # 多个参数分隔支持
  cid = "1|2|3"
  # 数组支持
  cid = [1,2,3]
  ```
- fields: 
  - 说明：读取指定字段，多个字段使用【|】分割
  - 是否必须：否
  - 默认值：空
- sons: 
  - 说明：是否读取下级栏目数据，默认为关闭，开启后读取当前栏目ID及下级栏目ID数据
  - 是否必须：否
  - 默认值：false
- no_cid
    - 说明：通过设置参数可排除不需要获取的栏目信息，可结合开启获取下级数据时使用，多个参数时使用 【|】分割
    - 是否必须：否
    - 默认值：空
    - 示例： ``no_cid="1|2|3"``
- limit
    - 说明: 获取指定数据条数，格式为 ``limit = "开始数,获取总数"``
    - 是否必须： 否
    - 默认值：10
    - 示例： `` limit = "100,10" `` 该示例从第100条数据开始，获取到第110条数据
- ids:
    - 说明：获取指定ID数据，多个数据使用【|】符号分割
    - 是否必须：否
    - 默认值：空
    - 示例： `` ids = 1 或 ids = "1|2|10" ``
- order
    - 说明：数据排序方式，
        - > 排序方式默认为倒序排列，多个排序字段时使用<font color="red">[,]（英文状态下逗号）</font>分割，通过 <font color="red">[|]（英文状态下中线）</font>分割排序类型
        - > ``desc`` 表示倒序排序(从大到小)， ``asc`` 表示正序排列(从小到大)
        - 预设类型
            - ``order = "new"`` 已发布时间倒序读取（最新排序）
            - ``order = "hot"`` 已访问量倒序读取（热点排序）
            - ``order = "updated_at"`` 已更新时间倒序读取
            - ``order = "created_at"`` 已创建时间倒序读取
            - ``order = "sort"`` 已手动设置的排序号倒序读取
            - ``order = "id|desc, created_at|asc"`` 自定义设置排序方式
            - > 上一案例表示已ID倒序排列，并按照创建时间正序排列，默认情况下按照倒序获取数据，案例可修改为 ``order = "id, created_at|asc"``
    - 是否必须：否
    - 默认值：``order = 'id|desc'``
- keyword【暂时取消】
    - 说明：搜索关键词多个关键词使用 【|】分割
    - 是否必须：否
    - 默认值：空
- m_id
    - 说明：模型ID
    - 是否必须：否
    - 默认值：空
- flag
    - 说明：包含的推荐属性
    - 是否必须：否
    - 默认值：空
- no_flag
    - 说明：不包含的推荐属性
    - 是否必须：否
    - 默认值：空
- cover
    - 说明：封面是否必须，只获取存在封面的的图片
    - 是否必须：否
    - 默认值：空
- rank【暂取消】
    - 说明：是否限制权限获取
    - 是否必须：否
    - 默认值：空
- with 
  - 说明：关联数据加载
  - 是否必须：否
  - 默认值：空
  - 示例： ``with = "category:id,name|extend:id,name"``
